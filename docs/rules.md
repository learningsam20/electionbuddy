# Project Guidelines

1. What you have to submit
   Your submission should include:

A link to your deployed app on the web
A link to your public GitHub repository (or Cloud Run deployment link)
README — must explain:
Chosen vertical and problem area
Core features and user flows
Use of Google Services
Any assumptions
Approach and logic
How the solution works

Evaluation: Hackathon judges will review your submission based on:
Completeness & Functionality
Use of Google Cloud Services
Creativity & Innovation
UI/UX Quality

2. Judging Criteria

Innovation - must be innovative, first of its kind, easy to use
Code Quality – structure, readability, maintainability
Security – safe and responsible implementation
Efficiency – optimal use of resources
Testing – validation of functionality
Accessibility – inclusive and usable design
Google Services – meaningful integration of Google Services
